# 外部設計（画面・DB・API）作成の振り返り

## 実施内容
要件定義をベースに、開発に必要な具体的な設計図を作成しました。

### 1. 画面設計 (Wireframes)
- `Docs/design/wireframes.md` を更新。
- タイムラインに「残り時間」を表示するバッジや、3つのカテゴリーを選択する投稿画面の構成を定義しました。
- Mermaidを使用して簡易的な画面遷移図を追加しました。

### 2. データベース設計 (DB Schema)
- `Docs/db/schema.md` を更新。
- `Posts` テーブルに、投稿の寿命を管理するための `lifespan_category` と `expires_at` カラムを追加。
- 削除対象を高速に検索するためのインデックス設計を明記しました。

### 3. API設計 (API Design)
- `Docs/api/endpoints.md` を新規作成。
- 投稿の作成と一覧取得のためのリクエスト/レスポンス形式を定義。
- フロントエンドでカウントダウンを表示しやすいよう、レスポンスに `remaining_seconds` を含める設計にしました。

## 次のステップ
設計が完了したため、次はいよいよ**「環境構築」**または**「バックエンド・フロントエンドの実装」**に進むことができます。
まずはデータベースのセットアップから開始することをお勧めします。
