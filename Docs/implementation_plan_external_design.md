# 実装計画：外部設計（画面設計・データベース設計）の作成

## 1. 目的
要件定義書の内容を具体的なシステム構成に落とし込み、開発の土台を作る。

## 2. タスク概要
1. **画面設計（ワイヤーフレーム）の作成**
    - 投稿一覧（タイムライン）
    - 寿命設定付きの投稿作成画面
2. **データベース設計の作成**
    - 投稿データと寿命管理のためのテーブル設計
3. **API設計の作成**
    - フロントエンドとバックエンド間の通信定義

## 3. 詳細手順

### Step 1: 画面設計 (`Docs/design/wireframes.md`)
- タイムライン：投稿カードに「残り時間」を表示するデザインを定義。
- 投稿作成：3つのカテゴリー（感情のデトックス、インスタント共有、議論・FB）を選択するUIを定義。

### Step 2: データベース設計 (`Docs/db/schema.md`)
- `Posts` テーブル：`content`, `lifespan_type`, `expires_at` などのカラムを定義。
- `Users` テーブル：基本的なユーザー情報を定義。

### Step 3: API設計 (`Docs/api/endpoints.md`)
- `POST /api/posts`：新規投稿作成
- `GET /api/posts`：有効な投稿の一覧取得

## 4. リスク・留意点
- 削除の遅延が発生しないよう、`expires_at` のインデックス設計を考慮する。
- UIは汎用的なパーツ（Shadcn UI等）を想定して設計する。
