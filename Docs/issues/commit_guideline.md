# コミットメッセージ・ガイドライン

本プロジェクトにおいて、変更履歴を明確にし、チームメンバーが変更内容を即座に理解できるようにするためのルールを定めます。

## 1. コミットメッセージの基本構成

原則として、以下の形式で記述してください。

```text
<type>: <subject>

<body> (任意)
```

## 2. Type の種類と意味

| Type | 意味・使用場面 |
| :--- | :--- |
| feat | 新機能の追加（例：投稿の寿命選択機能の実装） |
| fix | バグ修正（例：カウントダウンの計算ロジック修正） |
| docs | ドキュメントのみの修正（例：README, docs内の修正） |
| style | コードの意味に影響を与えない変更（改行、セミコロン、フォーマット等） |
| refactor | バグ修正も機能追加も行わないコードの整理 |
| perf | パフォーマンス向上のための修正 |
| test | テストコードの追加・修正 |
| chore | ビルドプロセスやライブラリの更新（例：NuGetパッケージの追加） |
